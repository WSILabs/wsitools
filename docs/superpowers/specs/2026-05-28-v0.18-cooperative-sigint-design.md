# wsitools v0.18 — Cooperative SIGINT shutdown for convert DZI/SZI

**Status:** Design approved 2026-05-28.

## 1. Goal

Make `convert --to dzi|szi` exit cleanly when the user sends SIGINT
(Ctrl-C) or any signal that cancels `cmd.Context()`. v0.17 left
`TestConvertDZICtxCancel` skipped because the descent stage's channel
send blocks indefinitely once encoder workers exit on `ctx.Done()`,
leaving the pipeline stuck.

After v0.18, SIGINT on an in-progress convert produces a process
exit within ~500 ms, and `TestConvertDZICtxCancel` passes without
its 10-second SIGKILL backstop firing.

## 2. Scope

In scope:
- Add `ctx context.Context` field to `levelBuilder` in
  `cmd/wsitools/convert_dzi_descent.go`.
- Propagate the descent's ctx into every builder during chain
  construction in `runDescent`.
- Make `levelBuilder.emitRow`'s channel send selectable on
  `ctx.Done()`. On cancel, release the pooled RGB tile buffer back
  to the pool and return.
- Unblock cascade unwinding: `feed`, `acceptDownsampled`, and
  `flush` continue calling each other directly (no further channel
  sends), so cancel propagates via early returns from `emitRow`.
- Re-enable `TestConvertDZICtxCancel` (was `t.Skip`'d in v0.17).
- New unit test `TestLevelBuilderEmitRowRespectsContext`.

Out of scope:
- Partial-output preservation. A cancelled convert leaves a partial
  DZI tile tree on disk; the user is expected to delete + retry.
  Cleanup would require additional plumbing into the sink drainer
  and dirFS.
- Manifest writing on cancel. v0.17 writes the DZI manifest only
  at successful `dzi.Writer.Close()`. On cancel, manifest is
  absent; consumers treat the partial directory as invalid. Fine.
- Graceful "drain in-flight tiles before exit." We accept that
  some encoder workers may be mid-cgo when cancel fires; they
  finish that one tile and then exit. No timeout on the in-flight
  work.

Non-goals:
- Changes to ScaledStrips' cancel behavior (already correct via
  `WithStripContext`).
- Changes to the encoder pool's cancel behavior (already correct
  via the worker's outer `select` on `ctx.Done()`).
- Changes to the sink drainer (already correct — exits when
  `writeJobs` is closed by `runDescent`).

## 3. Architecture

### 3.1 Current pipeline cancel paths

`runDescent` (v0.17) lays out three concurrent stages:

1. **ScaledStrips iterator** (drives `runDescent`'s main goroutine):
   already cancel-aware via `WithStripContext(ctx)`. `it.Next()`
   returns an error after cancel.
2. **Descent + level builders** (same main goroutine, called
   between `it.Next()` invocations): NOT cancel-aware. The channel
   send in `emitRow` is unconditional.
3. **Encoder workers** (N goroutines): cancel-aware. Each worker's
   outer loop selects on `ctx.Done()`.
4. **Sink drainer** (1 goroutine): ranges over `writeJobs`; exits
   when the channel is closed.

The failure mode in v0.17:

```
SIGINT fires
  → ctx.Done() closes
  → encoder workers exit on next loop iteration
  → encodeJobs channel fills (workers stopped draining)
  → emitRow blocks on `lb.jobs <- encodeJob{...}`
  → runDescent's main goroutine is wedged inside top.feed(strip)
  → it.Next() is never called again
  → no opportunity to detect cancel and break out
  → cmd.Wait() hangs in the caller
```

### 3.2 Fix

Add `ctx context.Context` to `levelBuilder`:

```go
type levelBuilder struct {
    level int
    width int
    cfg   dzi.Config
    cols  int
    rows  int

    prev, cur, next *rgbImage
    accum     *rgbImage
    accumRows int
    rowIndex  int

    child *levelBuilder
    jobs  chan<- encodeJob
    ctx   context.Context // NEW: cancellation signal for emitRow's send
}
```

In `runDescent`, pass `ctx` when constructing each builder:

```go
lb := &levelBuilder{
    level: lvl, width: lw, cfg: cfg,
    cols: cols, rows: rows,
    jobs: encodeJobs,
    ctx:  ctx, // NEW
}
```

Make `emitRow`'s send cancel-aware:

```go
func (lb *levelBuilder) emitRow(row int) {
    for col := 0; col < lb.cols; col++ {
        tile := lb.assembleTile(col, row)
        select {
        case lb.jobs <- encodeJob{level: lb.level, col: col, row: row, img: tile}:
        case <-lb.ctx.Done():
            releaseRGB(tile)
            return
        }
    }
}
```

On cancel, the loop returns without enqueuing remaining cols. The
caller (`feed` or `flush`) returns without further action because
their work is just calling `emitRow` + `child.acceptDownsampled` /
`child.flush`. The cascade unwinds entirely via Go's normal return
path.

### 3.3 Buffer-leak prevention

`assembleTile` borrows a pooled RGB slice via `newPooledRGB`. The
encoder normally returns it to the pool via `releaseRGB(img)` inside
`encodeTile`. On cancel, the tile never reaches the encoder, so
`emitRow` must release the buffer itself — line `releaseRGB(tile)`
in the `<-lb.ctx.Done()` arm above.

No other allocations leak. Strip-level rolling buffers (`prev`,
`cur`, `next`) are short-lived; they get GC'd after the goroutine
returns.

### 3.4 runDescent main-loop interaction

After the fix, when SIGINT fires mid-`top.feed(strip)`:

1. The descent cascade is processing tile (col, row). emitRow
   selects on `ctx.Done()` and returns.
2. The cascade unwinds back to `runDescent`'s main loop.
3. Next iteration: `it.Next()` returns `context.Canceled` (the
   iterator was already cancelled via `WithStripContext`).
4. Main loop sets `srcErr = err`, calls `cancel()` (idempotent),
   breaks.
5. `close(encodeJobs)` — but workers may already have exited on
   `ctx.Done()`; this is harmless.
6. `encWG.Wait()` returns (workers are gone).
7. `close(writeJobs)` — sink drains remaining writes.
8. `sinkWG.Wait()` returns.
9. `runDescent` returns `srcErr`.
10. Caller (`runConvertDZI`) propagates the error.
11. `main()` exits with code 130 (the existing `errors.Is(err,
    context.Canceled)` handler).

Total elapsed wall-time from SIGINT to process exit: bounded by
the slowest encoder worker finishing its current cgo encode call
(libjpeg-turbo doesn't honor signals mid-encode). ~5-20 ms per
tile × the worst-case in-flight count (workers + channel buffer)
≈ ~100-500 ms.

## 5. Test plan

### 5.1 New unit test

`TestLevelBuilderEmitRowRespectsContext` in
`cmd/wsitools/convert_dzi_descent_test.go`:

- Construct a `levelBuilder` with a small `cur` strip and a
  cancelled `context.Context`.
- Call `lb.emitRow(0)`.
- Assert: returns within 1 ms (no send blocking), and no
  `encodeJob` arrives on `lb.jobs` (the channel is empty/blocked).
- Assert: the pooled RGB buffer was returned (count pool size
  before/after, or use a custom probe channel).

Pool-return verification is the tricky bit. Simplest approach:
size the `jobs` channel to capacity 0 (unbuffered), then a normal
send would block; if the test completes without timeout, we know
the cancel path was taken.

### 5.2 Re-enable existing test

Remove the `t.Skip(...)` from `TestConvertDZICtxCancel`. The
existing body:

```go
cmd := exec.Command(findBinary(t), "convert", "--to", "dzi", "-o", out, in)
cmd.Start()
time.AfterFunc(100*time.Millisecond, func() {
    cmd.Process.Signal(os.Interrupt)
})
killed := false
killTimer := time.AfterFunc(10*time.Second, func() {
    killed = true
    cmd.Process.Kill()
})
cmd.Wait()
killTimer.Stop()
if killed {
    t.Errorf("convert did not exit within 10s of SIGINT; required SIGKILL")
}
```

With v0.18, `killed` stays `false` because SIGINT now unwinds in
~100-500 ms. Test passes.

### 5.3 Race-detector run

`go test -race ./cmd/wsitools -run 'TestLevelBuilder|TestConvertDZI'`
catches any race introduced by storing ctx on the builder + reading
it from `emitRow`. Expected to be clean — ctx is set at
construction and only read thereafter.

### 5.4 Acceptance gate

- All unit tests pass.
- `TestConvertDZICtxCancel` passes in <2 s (was skipping; now runs).
- All existing integration tests continue to pass.
