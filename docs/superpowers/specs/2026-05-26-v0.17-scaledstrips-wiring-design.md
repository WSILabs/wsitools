# wsitools v0.17 — ScaledStrips Wiring for `convert --to dzi|szi`

**Status:** Design approved 2026-05-26.

## 1. Goal

Replace `emitDZIPyramid`'s per-tile `slide.ReadRegionScaled` calls
with opentile-go v0.26's `slide.ScaledStrips` iterator. The iterator
amortizes source-tile decode work across all DZI tiles in a
horizontal strip via an internal per-iterator tile cache, and runs
decode workers in parallel. On striped sources (NDPI, OME-OneFrame)
this turns DZI/SZI rendering from "correct but unusably slow"
(>5min on CMU-1.ndpi, timing out in CI) into a workable runtime.
On natively-tiled sources the gain is more modest (parallelism +
cache amortization) but still positive.

After v0.17, the previously-skipped `TestConvertNDPIToDZI` and
`TestConvertNDPIToSZI` integration tests run under a 60-second
timeout in CI.

## 2. Scope

In scope for v0.17:

- Rewrite `emitDZIPyramid` (`cmd/wsitools/convert_dzi.go`) to
  dispatch each DZI level through one `*StripIterator` per level,
  with a 3-strip rolling buffer for overlap handling.
- Keep `renderDZITile`'s per-tile path as a fallback for levels
  whose strip count would be ≤1 (DZI levels where output height is
  ≤ tile size; the coarsest few levels of every pyramid).
- Reuse the existing `--workers N` flag to control ScaledStrips'
  decode worker count via `WithStripWorkers(n)`.
- Propagate `cmd.Context()` into the iterator via
  `WithStripContext` for cancellation.
- New unit test: pure-function test of the per-tile assembly logic.
- Re-enable previously-skipped NDPI integration tests with a 60s
  timeout.
- Cancellation test verifying clean goroutine shutdown.

Out of scope (deferred):

- **Parallel raw-tile fetch + decode for `--to svs|tiff|ome-tiff
  --codec X` on striped sources.** Different optimization; tracked
  on `docs/roadmap.md` under Architectural > Deferred from v0.17.
- **libvips `dzsave` benchmark harness.** Was named alongside
  ScaledStrips in v0.16's CHANGELOG; separate design once v0.17
  perf is real.
- **Encoder-worker parallelism for DZI tile encode.** Add only if
  v0.17 profiling shows tile encoding is the next bottleneck.
- **Auto-tuned per-level worker count.** Use the single
  `--workers` knob.
- **New CLI flags** for strip height, kernel, IDCT scale, or
  lookahead. ScaledStrips' defaults (workers = NumCPU, lookahead =
  2, IDCT scale = auto, kernel = Lanczos) are correct for every
  realistic configuration.
- **Multi-image OME sources.** ScaledStrips uses `imageIdx=0` only;
  multi-image OME is its own track.

Non-goals:

- Changing DZI / SZI output format or detection semantics.
- Changing pixel-output for natively-tiled sources beyond what
  resample-kernel differences may introduce at strip seams. The
  existing `TestConvertToDZI` / `TestConvertToSZI` assert
  structural correctness, not byte/pixel-exact output, so the
  switchover passes without modification.

## 3. Architecture

### 3.1 Level dispatch

```go
func emitDZIPyramid(ctx context.Context, slide *opentile.Slide, w dziTileSink, cfg dzi.Config) error {
    max := dzi.MaxLevel(cfg.Width, cfg.Height)
    quality := parseDZIQuality(cvQuality)
    workers := cvWorkers
    for lvl := max; lvl >= 0; lvl-- {
        lw, lh := dzi.LevelDims(cfg.Width, cfg.Height, lvl)
        cols, rows := dzi.GridDims(lw, lh, cfg.TileSize)
        if rows <= 1 {
            if err := emitLevelPerTile(ctx, slide, w, cfg, lvl, lw, lh, cols, rows, quality); err != nil {
                return err
            }
            continue
        }
        if err := emitLevelViaStrips(ctx, slide, w, cfg, lvl, lw, lh, cols, rows, workers, quality); err != nil {
            return err
        }
    }
    return nil
}
```

The `rows <= 1` fallback covers DZI levels where LH ≤ TileSize.
With the default TileSize=256, that's level 0..8 (LH = 1, 2, 4, 8,
16, 32, 64, 128, 256). For a 50000-tall slide whose MaxLevel is
~16, that's 9 of 17 levels using the per-tile path. Per-tile is
cheap at those sizes; ScaledStrips' worker-pool spin-up overhead
would not amortize.

### 3.2 Strip-based level emission

```go
func emitLevelViaStrips(ctx context.Context, slide *opentile.Slide, w dziTileSink, cfg dzi.Config,
    lvl, lw, lh, cols, rows, workers, quality int) error {

    it := slide.ScaledStrips(
        image.Rect(0, 0, cfg.Width, cfg.Height),
        image.Point{X: lw, Y: lh},
        cfg.TileSize,
        opentile.WithStripContext(ctx),
        opentile.WithStripWorkers(workers),
    )
    defer it.Close()

    var prev, cur *decoder.Image
    cur, err := it.Next()
    if err != nil {
        return fmt.Errorf("level %d strip 0: %w", lvl, err)
    }

    for row := 0; row < rows; row++ {
        var next *decoder.Image
        if row+1 < rows {
            next, err = it.Next()
            if err != nil {
                return fmt.Errorf("level %d strip %d: %w", lvl, row+1, err)
            }
        }
        for col := 0; col < cols; col++ {
            body, err := encodeDZITileFromStrips(prev, cur, next, cfg, lvl, lw, lh, col, row, quality)
            if err != nil {
                return fmt.Errorf("level %d (%d,%d): %w", lvl, col, row, err)
            }
            if err := w.WriteTile(lvl, col, row, body); err != nil {
                return err
            }
        }
        prev = cur
        cur = next
    }
    return nil
}
```

### 3.3 Per-tile assembly from strips

`encodeDZITileFromStrips` is the new pure-ish function that
assembles a DZI tile from up to three strip images plus the
existing overlap-bounds math. Signature:

```go
func encodeDZITileFromStrips(
    prev, cur, next *decoder.Image,
    cfg dzi.Config,
    lvl, lw, lh, col, row, quality int,
) ([]byte, error)
```

Logic:

1. Compute `cw, ch` (content dims) via `dzi.EdgeTileDims`.
2. Compute `leftOv, rightOv, topOv, bottomOv` per existing
   suppress-at-image-edge rules.
3. Allocate `image.RGBA` destination of size
   `(cw + leftOv + rightOv) × (ch + topOv + bottomOv)`.
4. Copy three vertical bands into the destination:
   - From `prev` (bottom `topOv` rows): only if `topOv > 0` and
     `prev != nil`.
   - From `cur` (all `ch` content rows): always.
   - From `next` (top `bottomOv` rows): only if `bottomOv > 0` and
     `next != nil`.
5. For each band, the source x-range is
   `[col*cfg.TileSize - leftOv, col*cfg.TileSize - leftOv + outW)`
   (clipped at strip edges; left overlap is suppressed at col=0).
6. RGB→RGBA copy is a manual loop (alpha=0xFF), mirroring
   `decoderToStdImage`'s body for the RGB-source case.
7. Encode via `jpeg.Encode` or `png.Encode` per `cfg.Format`.

The `cur` strip's row range is `[0, ch)` — for the last DZI row of
an image whose `LH` isn't a multiple of `TileSize`, the strip is
short and `ch` matches the strip's actual height. No special case.

### 3.4 Worker count

`--workers N` reuses `cvWorkers` (already defined in
`cmd/wsitools/convert.go` from v0.16). For DZI/SZI:

- `cvWorkers > 0` → passed to `WithStripWorkers(cvWorkers)`.
- `cvWorkers == 0` → ScaledStrips' default (GOMAXPROCS), achieved
  by passing the option only when `cvWorkers > 0`.

### 3.5 Cancellation

`cmd.Context()` is propagated via `WithStripContext`. The iterator's
worker pool and lookahead goroutine derive a cancellable context
from it; on cancel, in-flight decodes return promptly and `Close`
joins cleanly.

The outer level-dispatch loop also checks `ctx.Err()` between
levels.

## 4. File layout

**Modified:**

**New:**

**Unchanged:**

- `cmd/wsitools/convert_szi.go` — calls `emitDZIPyramid` via the
  same `dziTileSink` interface.
- `cmd/wsitools/convert.go` — no new flags.
- `internal/dzi/`, `internal/szi/` — no writer-side changes.

## 5. Pipeline reuse

`internal/pipeline` (the worker-pool decode/process/encode runner)
is **not** used for v0.17. The TIFF re-encode path uses it; the
DZI/SZI path uses ScaledStrips directly. ScaledStrips owns its
internal worker pool — adding another pipeline layer on top would
serialize the strip yield path and prevent the iterator's
overlapping-decode-and-yield design from working.

## 6. Memory profile

Three RGB strips in RAM at peak: `prev`, `cur`, `next`. Per strip
at a DZI level with width LW and stripHeight TileSize:

- LW × TileSize × 3 bytes.

Worst case: L0 of a large slide. CMU-1.ndpi has L0 ≈ 50000 × 32500.
At TileSize=256, one strip is 50000 × 256 × 3 ≈ 36 MB. Three strips
≈ 108 MB peak. Acceptable on any reasonable machine.

For `--dzi-tile-size 512` (allowed but not default), the peak is
~216 MB. Still acceptable.

Per-iterator tile cache memory is bounded by the iterator's own
LRU policy (opentile-go v0.26's design) and is separate from the
strips wsitools holds.

## 7. Test plan

### 7.1 Unit tests (no fixtures)

`cmd/wsitools/convert_dzi_strip_test.go`:

- **Synthetic strip pattern.** Build three `*decoder.Image` strips
  where each pixel encodes `(R, G, B) = (row_index, col_index,
  strip_index)`. Assert that `encodeDZITileFromStrips` produces a
  destination tile with pixels sourced from the right strip + row
  band + column range.
- **Corner case: row=0, col=0.** topOv=0, leftOv=0. Pixel (0,0)
  of destination = pixel (0,0) of `cur`. No `prev` strip read.
- **Corner case: bottom-right of image.** rightOv=0, bottomOv=0.
  No `next` strip read. Destination dimensions match short edge
  tile.
- **Interior tile.** All four overlaps active. Each of three
  source strips contributes the expected band/column range.
- **prev=nil safety.** Topmost DZI row passes `prev=nil`; function
  must not panic and must produce a tile with topOv=0.
- **next=nil safety.** Bottommost DZI row passes `next=nil`; same.

### 7.2 Integration tests (fixtures, gated by `WSI_TOOLS_TESTDIR`)

- **`TestConvertToDZI`** (existing) continues to pass on
  CMU-1-Small-Region.svs. Structural assertions only (manifest +
  tile tree); no pixel-exact comparison.
- **`TestConvertToSZI`** (existing) continues to pass.
- **`TestConvertNDPIToDZI`** — re-enable (was `t.Skip("v0.17:
  ...")`). 60-second timeout. Asserts the L0 tile of the resulting
  pyramid exists.
- **`TestConvertNDPIToSZI`** — re-enable with 60s timeout. Asserts
  the SZI archive opens and contains a `.dzi` manifest entry.
- **`TestConvertDZICtxCancel`** (new) — start a DZI convert in a
  goroutine with a context cancelled after 100ms; verify the call
  returns with `ctx.Err()` and no goroutines leak (use
  `runtime.NumGoroutine` before/after with a 100ms settle window).

### 7.3 Optional pixel-hash regression test (post-v0.17 follow-up)

`TestConvertToDZIPixelHashRegression` — convert
CMU-1-Small-Region.svs to DZI, decode every tile, SHA-256 the
concatenated decoded pixel bytes (sorted by `level,col,row` for
determinism). Compare against a golden value recorded once and
checked in. Catches silent resample/overlap regressions in future
changes. Not part of v0.17 itself — added as a follow-up commit
once the v0.17 pixel output is settled.

### 7.4 Acceptance gate

- `make test` — all unit tests pass.
- NDPI integration tests pass under 60s on a developer laptop
  (Apple Silicon). Linux CI may run slower but should fit under
  120s; if not, the design hasn't met its bar and we re-iterate
  before merge.
- Existing test matrix unchanged.
