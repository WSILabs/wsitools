# wsitools v0.17 — Pyramid-Descent DZI/SZI Generator + JPEG Codec Reorganization

**Status:** Design approved 2026-05-27.

## 1. Goal

Make `convert --to dzi|szi` competitive with libvips `dzsave` runtime
on real WSI sources, including striped formats like NDPI where the
v0.16 per-tile path takes >35 minutes. Achieved via three changes:

1. **Single descent through the source pyramid.** One `*StripIterator`
   opened at the largest DZI level's dimensions; coarser DZI levels
   built by 2× downsample cascade in process, so source-tile decode
   work is shared across all DZI levels (instead of redecoded once
   per qualifying level as in v0.17's first attempt).

2. **Parallel JPEG encoder pool.** Multiple worker goroutines encode
   DZI tiles concurrently. Sink writes are serialized through a
   single goroutine so SZI's `archive/zip.Writer` (not
   concurrent-safe) sees ordered writes.

3. **libjpeg-turbo-backed vanilla JPEG codec.** wsitools'
   `internal/codec/jpeg` is reorganized from "Aperio APP14+raw-RGB
   quirk" into vanilla YCbCr+4:2:0 standalone JPEGs. The
   Aperio-specific encoder is preserved as `internal/codec/aperioapp14`
   (not Factory-registered; for forensic emulation only). All
   wsitools `--codec jpeg` outputs (DZI/SZI/TIFF/OME-TIFF/SVS) now
   emit vanilla JPEGs — matching Grundium's third-party SVS
   convention and what every consumer (browsers, openslide, libvips,
   QuPath, opentile-go) reads natively.

The previous v0.17 attempt (per-DZI-level iterator) was abandoned;
see `~/.claude/projects/.../memory/project_v0.17_failed_attempt.md`
for why.

## 2. Scope

**In scope for v0.17:**

- New pyramid-descent renderer in `cmd/wsitools/convert_dzi.go`:
  - Single `*StripIterator` consuming source at L_max DZI dims.
  - Per-level `levelBuilder` struct holding a rolling 3-strip RGBA
    buffer + a partial-strip accumulator.
  - Box-filter 2× downsample between levels; cascade from L_max
    down through L_0.
  - All DZI levels uniformly handled by the cascade (no per-tile
    fallback for coarse levels; the cascade naturally handles
    them).
- Parallel encoder pool calling vanilla `jpeg.Factory` (configured
  via `--workers`).
- Single-goroutine sink drain calling `dziTileSink.WriteTile`
  in arrival order.
- JPEG codec reorganization:
  - `internal/codec/jpeg/` rewritten to produce vanilla YCbCr+4:2:0
    JPEGs (no APP14, no raw-RGB colorspace).
  - `internal/codec/aperioapp14/` new package containing the
    previous Aperio APP14 encoder as a direct (non-Factory) type,
    with zero current callers, preserved for future use.
  - SVS-output paths in `cmd/wsitools/convert_tiff.go` stop using
    APP14 encoding — produce vanilla JPEGs that pass Aperio
    detection via ImageDescription "Aperio" prefix (same approach
    Grundium uses).
- libvips parity benchmark: `scripts/bench-dzi.sh` + `make bench-dzi`
  target running `wsitools convert --to dzi` and `vips dzsave` on
  the same fixtures, printing a runtime ratio table.
- Re-enable the v0.16-skipped NDPI integration tests
  (`TestConvertNDPIToDZI`, `TestConvertNDPIToSZI`) with a
  120-second timeout.
- New `TestConvertDZICtxCancel` cancellation test.
- Unit tests for `levelBuilder` downsample math, cascade flow,
  encoder pool concurrency, vanilla `jpeg.Factory` output decoding
  cleanly via stdlib `image/jpeg`.

**Out of scope (deferred):**

- **Parallel raw-tile fetch + decode for `--to svs|tiff|ome-tiff
  --codec X` on striped sources.** Tracked under
  `docs/roadmap.md` > Architectural > Deferred from v0.17. Different
  optimization (no scaling, parallel pipeline reuse).
- **Permanent CI gate against libvips.** `make bench-dzi` is
  manual; CI doesn't require libvips installed. Adding a flake-
  tolerant CI comparison is a separate concern.
- **Multi-image OME sources.** ScaledStrips uses `imageIdx=0`;
  multi-image OME is its own track.
- **Lanczos downsample for the cascade.** Box filter is fine for
  DZI pyramid generation (matches libvips' dzsave default). Lanczos
  is reserved for the ScaledStrips top-stage where source→target
  scaling matters most.
- **Deleting `internal/codec/aperioapp14/`.** Preserved as
  unused-but-buildable code. May be deleted in v0.18+ if nothing
  picks it up.
- **Pixel-exact parity** with v0.16 DZI output or with libvips
  dzsave output. Resample-kernel + JPEG-encoder differences make
  byte/pixel-equality intractable; equivalence is the goal.

**Non-goals:**

- Changes to the DZI / SZI on-disk format.
- Changes to opentile-go (consumed at v0.26+; no new dependencies).
- New CLI flags. `--workers` controls both decode and encode worker
  counts.

## 3. Architecture

### 3.1 Pipeline overview

Three concurrent stages connected by bounded channels.

```
┌───────────────┐  strips  ┌──────────────────┐  encodeJob  ┌──────────────┐  writeJob  ┌────────────┐
│ ScaledStrips  │ ───────► │ pyramid descent  │ ──────────► │ encoder pool │ ─────────► │ sink drain │ → DZI/SZI writer
│ iterator      │          │ + level builders │             │ (N workers)  │            │ (1 worker) │
└───────────────┘          └──────────────────┘             └──────────────┘            └────────────┘
```

**Stage 1: source reader.** One `*StripIterator` opened at
`outSize = (L_max_DZI_W, L_max_DZI_H) = (slide_L0_W, slide_L0_H)`,
`stripHeight = cfg.TileSize`, with `WithStripContext(ctx)` and
`WithStripWorkers(workers)`. ScaledStrips picks the source pyramid
level + IDCT scale via its existing `bestLevelForRegion` logic.
Yields strips of width `slide_L0_W` and height `cfg.TileSize` (last
strip possibly short).

**Stage 2: descent + level builders.** Single goroutine consumes
each strip from stage 1, then:

1. Hands the strip to the L_max `levelBuilder` (which holds the
   rolling buffer for that level).
2. The L_max builder slices the strip into DZI tiles for L_max and
   enqueues `encodeJob`s.
3. The L_max builder downsamples the strip 2× and feeds it into
   the L_max-1 builder. That builder may emit its own tiles +
   cascade further when it has enough rows.

**Stage 3: encoder pool.** N goroutines pull `encodeJob`s, call
`jpeg.Factory.Encode` (vanilla) or `png.Encode` (PNG case), enqueue
`writeJob`s. RGBA buffer ownership transferred from descent → encoder
worker; encoder frees the buffer (returns to a pool) after encoding.

**Stage 4: sink drain.** Single goroutine pulls `writeJob`s in
arrival order and calls `dziTileSink.WriteTile`. SZI's
`archive/zip.Writer` is not concurrent-safe; single-goroutine
serialization handles this. Tile write order is unrestricted by
DZI/SZI semantics (each tile is an independent file/zip entry).

### 3.2 Level builder

```go
type levelBuilder struct {
    level     int
    width     int                    // level pixel width
    cfg       dzi.Config
    cols      int                    // tile column count at this level
    rows      int                    // tile row count at this level

    // Rolling 3-strip overlap buffer (RGBA, each width×TileSize).
    prev, cur, next *image.RGBA

    // Accumulator for the strip currently being filled from
    // downsampled parent rows.
    accum     *image.RGBA            // width × TileSize
    accumRows int                    // how many rows of accum are filled

    rowIndex  int                    // next DZI row to emit (0 to rows)

    // Cascade target (one level coarser); nil at coarsest level.
    child     *levelBuilder

    // Output queue (shared across all builders).
    jobs      chan<- encodeJob
}
```

**Method: `feed(parentStrip *image.RGBA)`.** Called by the parent
(or by the descent goroutine for L_max).

For the L_max builder, the input strip width equals `width` —
slice directly into DZI tiles:

1. Rotate buffer: `prev = cur; cur = next; next = parentStrip`.
2. If `cur != nil` (we have at least one complete prior strip plus
   a successor for bottom overlap): emit DZI tiles for row
   `rowIndex` by slicing `cur` with overlap from `prev`/`next`.
   Increment `rowIndex`.
3. Cascade: produce a `width/2 × TileSize/2` downsampled copy of
   `parentStrip` via `boxDownsample2x` and call `child.feed(...)`
   with this half-strip.

For non-L_max builders, the input is a half-height "child strip"
from the parent. The builder accumulates these into `accum` until
`accumRows == TileSize`, then promotes `accum` to a full strip and
behaves as L_max does (rotate buffer, emit, cascade).

**Method: `flush()`.** Called when the descent finishes. Emits any
remaining tiles whose `prev`/`cur`/`next` rolling buffer hasn't been
fully drained, including the last DZI row at this level (which has
no `next` strip). Then calls `child.flush()`.

**Box downsample.**

```go
func boxDownsample2x(src *image.RGBA) *image.RGBA {
    w := src.Bounds().Dx() / 2
    h := src.Bounds().Dy() / 2
    dst := image.NewRGBA(image.Rect(0, 0, w, h))
    for y := 0; y < h; y++ {
        for x := 0; x < w; x++ {
            sx, sy := x*2, y*2
            r := (uint32(src.Pix[(sy)*src.Stride+sx*4+0]) +
                  uint32(src.Pix[(sy)*src.Stride+(sx+1)*4+0]) +
                  uint32(src.Pix[(sy+1)*src.Stride+sx*4+0]) +
                  uint32(src.Pix[(sy+1)*src.Stride+(sx+1)*4+0])) / 4
            // ...same for G, B...
            dst.Pix[y*dst.Stride+x*4+0] = byte(r)
            // ...
            dst.Pix[y*dst.Stride+x*4+3] = 0xFF
        }
    }
    return dst
}
```

(Alpha is constant 0xFF; only RGB averaged. Hot loop; benchmark
against SIMD-vectorized variant in v0.18 if needed.)

**Memory footprint.** Each builder holds at most 4 RGBA strips
(prev/cur/next/accum), each `width × TileSize × 4 bytes`. For
CMU-1.ndpi (L_max width 51200), sum across all levels (51200 +
25600 + 12800 + … + 1) × 256 × 4 × 4 ≈ 200 MB peak. Acceptable.

### 3.3 Encoder pool

```go
type encodeJob struct {
    level, col, row int
    img             *image.RGBA   // owned by the encoder worker
}

type writeJob struct {
    level, col, row int
    body            []byte
}
```

N encoder goroutines pull from `encodeJobs`, encode, push to
`writeJobs`. The vanilla `jpeg.Factory` provides the encoder:

```go
factory := jpeg.Factory{}
enc, _ := factory.NewEncoder(codec.LevelGeometry{...}, codec.Quality{Q: cvQuality})
body, err := enc.EncodeStandalone(img.Pix, img.Bounds().Dx(), img.Bounds().Dy())
```

(`EncodeStandalone` is the new entry point — full self-contained
JPEG with embedded DQT/DHT, no APP14, YCbCr+4:2:0.)

For PNG, encoders call stdlib `image/png.Encode` directly. PNG
encode is rarely the bottleneck because PNG output is uncommon for
DZI/SZI viewers.

Image buffer reuse via `sync.Pool` of `*image.RGBA` is a v0.18
optimisation; v0.17 allocates per-tile.

### 3.4 Sink drain

```go
go func() {
    defer close(sinkDone)
    for job := range writeJobs {
        if err := sink.WriteTile(job.level, job.col, job.row, job.body); err != nil {
            errOnce.Do(func() { firstErr = err; cancel() })
            return
        }
    }
}()
```

Single goroutine. Handles both DZI's `dirFS` and SZI's
`zipFS` uniformly (the latter requires serial writes).

### 3.5 Cancellation + error propagation

A `context.CancelFunc` derived from `cmd.Context()` is closed on
the first error from any stage. Stages select on `ctx.Done()` and
exit promptly. The descent stage's strip-iterator gets the cancel
via `WithStripContext` propagation.

`sync.Once` tracks the first error so subsequent errors are not
overwritten. The orchestrating function waits for all stages via
`sync.WaitGroup` and returns the first error after drain.

### 3.6 Lifecycle

```go
func emitDZIPyramid(ctx context.Context, slide *opentile.Slide, sink dziTileSink, cfg dzi.Config) error {
    ctx, cancel := context.WithCancel(ctx)
    defer cancel()

    workers := cvWorkers
    if workers <= 0 {
        workers = runtime.NumCPU()
    }

    encodeJobs := make(chan encodeJob, 2*workers)
    writeJobs  := make(chan writeJob, 2*workers)

    // Build level builders top-down.
    builders := buildLevelChain(cfg, encodeJobs)
    topBuilder := builders[len(builders)-1] // L_max

    // Spawn encoders.
    var encWG sync.WaitGroup
    for i := 0; i < workers; i++ {
        encWG.Add(1)
        go encoderWorker(ctx, encodeJobs, writeJobs, cfg, &encWG)
    }
    // Spawn sink drain.
    var sinkWG sync.WaitGroup
    sinkWG.Add(1)
    var sinkErr error
    go sinkDrainer(writeJobs, sink, &sinkErr, &sinkWG)

    // Drive source iteration.
    it := slide.ScaledStrips(...)
    defer it.Close()
    for {
        strip, err := it.Next()
        if err == io.EOF { break }
        if err != nil { /* propagate, cancel, drain, return */ }
        topBuilder.feed(strip)
    }
    topBuilder.flush()

    close(encodeJobs)
    encWG.Wait()
    close(writeJobs)
    sinkWG.Wait()

    return sinkErr // or first stage error
}
```

## 4. JPEG codec reorganization

### 4.1 Before / after layout

**Before (current state):**
```
internal/codec/jpeg/jpeg.go   Factory registered as "jpeg"
                              Encoder.EncodeTile → abbreviated JPEG with
                                APP14 raw-RGB marker (Aperio quirk).
                              Used by all transcode/convert re-encode paths.
                              WRONG for non-SVS containers (DZI/TIFF/OME-TIFF)
                              because most decoders ignore APP14 and render
                              hue-rotated.
```

**After:**
```
internal/codec/jpeg/jpeg.go   Factory registered as "jpeg"
                              Encoder.EncodeTile         → abbreviated, vanilla YCbCr.
                              Encoder.EncodeStandalone   → full self-contained JPEG, vanilla YCbCr.
                              Default chroma subsampling 4:2:0.
                              No APP14 marker.

internal/codec/aperioapp14/aperioapp14.go
                              AperioAPP14Encoder direct type (not Factory-registered).
                              Produces the previous APP14+raw-RGB output.
                              Zero current callers after this change.
                              Preserved for forensic emulation; may be deleted
                              in a future release if nothing picks it up.
```

### 4.2 C helper changes

The shared C helper `wsi_encode` in `internal/codec/jpeg/jpeg.go` is
duplicated into two variants in their respective packages:

- `jpeg/jpeg.go::wsi_encode_vanilla`:
  - `cinfo.in_color_space = JCS_RGB` (input pixels are RGB).
  - `jpeg_set_defaults(&cinfo)` — defaults to JCS_YCbCr output
    colorspace + 4:2:0 chroma subsampling.
  - No `jpeg_write_marker(JPEG_APP0+14, ...)` call.
  - `abbreviated` param controls whether DQT/DHT are written
    inline or omitted.

- `aperioapp14/aperioapp14.go::wsi_encode_aperio`:
  - `cinfo.in_color_space = JCS_RGB`.
  - `jpeg_set_colorspace(&cinfo, JCS_RGB)` — overrides default;
    output stored as RGB (no YCbCr conversion).
  - `jpeg_write_marker(JPEG_APP0+14, adobeAPP14Payload, 14)` after
    `jpeg_start_compress`.
  - `abbreviated=true` always (this is the Aperio-tile format).

### 4.3 Call-site updates

- `cmd/wsitools/convert_tiff.go::runConvertTIFFReencode`: currently
  calls `codec.Lookup("jpeg")` which returned the Aperio encoder.
  After the reorg the Factory returns the vanilla encoder. SVS
  output (`container == "svs"`) now also produces vanilla JPEGs.
  This is the user-facing behaviour change.
- `cmd/wsitools/convert_dzi.go`: new code; uses vanilla
  `jpeg.Factory` via the encoder pool.
- `cmd/wsitools/convert_szi.go`: unchanged; reuses convert_dzi
  internals.

### 4.4 Tests for the codec reorg

- `internal/codec/jpeg/jpeg_test.go`:
  - Encode a known RGB pattern; decode via stdlib `image/jpeg`;
    assert the decoded pixels are close to input (within JPEG
    quantization noise at Q=85).
  - Encode with `EncodeStandalone`; verify output is a valid
    standalone JPEG (starts with `FF D8`, ends with `FF D9`,
    contains DQT and DHT markers, no APP14 marker).
  - Encode with `EncodeTile` (abbreviated); verify it has no DQT
    when combined with the tables from `LevelHeader()` it
    round-trips correctly via stdlib.
- `internal/codec/aperioapp14/aperioapp14_test.go`:
  - Encode a known pattern; verify APP14 marker is present + has
    Adobe identifier + ColorTransform=0.
  - Verify output decodes through libjpeg-turbo (via
    decoder/jpeg) — APP14-respecting decode produces input
    pixels.
  - Verify stdlib `image/jpeg` decode of the same bytes produces
    hue-rotated pixels (documenting the quirk).

### 6.3 libvips parity benchmark

`scripts/bench-dzi.sh`:

```bash
#!/usr/bin/env bash
# Run wsitools convert --to dzi and vips dzsave on the same fixtures,
# print runtime comparison. Manual; not in CI.

set -e
FIXTURES=(
    "sample_files/svs/CMU-1-Small-Region.svs"
    "sample_files/ndpi/CMU-1.ndpi"
    "sample_files/ndpi/OS-2.ndpi"
)
WSI=$(realpath bin/wsitools)
TMP=$(mktemp -d)
trap "rm -rf $TMP" EXIT

if ! command -v vips >/dev/null; then
    echo "vips not installed; skipping" >&2
    exit 0
fi

printf "%-40s %12s %12s %8s\n" "fixture" "wsitools(s)" "vips(s)" "ratio"
for fx in "${FIXTURES[@]}"; do
    [ -f "$fx" ] || continue
    name=$(basename "$fx")
    t1=$( { time -p "$WSI" convert --to dzi -o "$TMP/wsi-$name.dzi" "$fx" > /dev/null; } 2>&1 | awk '/real/{print $2}')
    t2=$( { time -p vips dzsave "$fx" "$TMP/vips-$name" --suffix '.jpeg[Q=85]' --tile-size 256 --overlap 1 > /dev/null; } 2>&1 | awk '/real/{print $2}')
    ratio=$(awk -v a="$t1" -v b="$t2" 'BEGIN{ if(b>0) printf "%.2fx", a/b; else print "n/a" }')
    printf "%-40s %12s %12s %8s\n" "$name" "$t1" "$t2" "$ratio"
done
```

`Makefile` addition:

```makefile
.PHONY: bench-dzi
bench-dzi: build
	@scripts/bench-dzi.sh
```

Result table goes into the v0.17 release notes verbatim.
