# wsitools v0.16 — `convert` Unification + DZI / SZI Targets

**Status:** Design approved 2026-05-25.

## 1. Goal

Collapse `transcode` into `convert`. A single `convert --to <target>`
subcommand becomes the universal output dispatcher. Add `dzi` and
`szi` as new targets so wsitools can produce OpenSeadragon-compatible
pyramids and Smart Zoom Image archives directly.

After v0.16, every output container is reached through one
subcommand with a uniform flag surface; `transcode` is hard-removed.

## 2. Scope

In scope for v0.16:

- New `internal/dzi/` writer package: tile-tree + manifest XML.
- New `internal/szi/` writer package: DZI tree + uncompressed-stored
  ZIP wrapper + optional `scan-properties.xml` from source metadata.
- `cmd/wsitools/convert.go` rewritten as universal dispatcher with
  `--to <cog-wsi|svs|tiff|ome-tiff|dzi|szi>`.
- Per-target subcommand files:
  `convert_cogwsi.go` (existing, refactored),
  `convert_tiff.go` (svs / tiff / ome-tiff, ported from `transcode.go`),
  `convert_dzi.go` (new),
  `convert_szi.go` (new, delegates to dzi writer + ZIP).
- `cmd/wsitools/transcode.go` and `transcode_test.go` DELETED.
- Existing pipeline (`internal/pipeline`) is reused unchanged; only
  the sink differs per target.
- Tile-copy-vs-re-encode decision generalised: any target whose
  container can hold the source's tile bytes verbatim + no `--codec`
  requested + source is natively-tiled → tile-copy. Otherwise
  re-encode. DZI / SZI never tile-copy (overlap + extra pyramid
  levels required).
- Integration tests: per-target, exercising tile-copy and re-encode
  paths where applicable.
- Documentation: roadmap.md shipped section, CHANGELOG.md, README
  update if it lists subcommands.

Out of scope (deferred):

- `--to dicom-wsi` — separate writer; tracked on roadmap as a
  larger item.
- Iris-codec write support — unrelated to SZI (SZI uses JPEG / PNG);
  remains deferred independently.
- ScaledStrips wiring into the convert pipeline — v0.17.
- libvips dzsave performance-comparison harness — v0.17.
- Narrow tile-copy path for DZI when `--dzi-overlap 0` + matching
  tile-size + matching codec — possible future optimisation; not
  worth the code path until benchmarks justify it.
- SZI: vendor metadata round-trip beyond passthrough; the `vendor/`
  folder is not populated by wsitools v0.16.

Non-goals:

- Performance optimisation of any existing target.
- Changing the bit-exact tile-copy promise for cog-wsi from
  natively-tiled sources (v0.6).
- Output codec additions (jpegli, HEIF, etc. remain deferred per
  roadmap).

## 3. Surface

```
wsitools convert --to <target> [flags] INPUT OUTPUT
```

**Targets:**

| `--to`     | Output                                  | Tile-copy eligible?      |
|------------|-----------------------------------------|--------------------------|
| `cog-wsi`  | COG-WSI (.tif / .cog)                   | yes (default)            |
| `svs`      | Aperio SVS pyramid                      | yes (no `--codec` + tiled src) |
| `tiff`     | Generic tiled TIFF pyramid              | yes (no `--codec` + tiled src) |
| `ome-tiff` | OME-TIFF pyramid                        | yes (no `--codec` + tiled src) |
| `dzi`      | DeepZoom pyramid (dir + manifest)       | never (overlap + pyramid depth) |
| `szi`      | Smart Zoom Image (DZI + ZIP)            | never (same reason as DZI)      |

**Common flags (all targets):**
- `--workers <n>` — pipeline worker count (existing default rule).

**Re-encode flags for `cog-wsi` / `svs` / `tiff` / `ome-tiff`:**
- `--codec <jpeg|jpegxl|avif|webp|htj2k>` — output tile codec.
  Presence triggers re-encode; absence triggers tile-copy when
  eligible. Not applicable to `dzi` / `szi` (use `--dzi-format`);
  passing `--codec` with `--to dzi` or `--to szi` errors.
- `--quality <int>` — codec-specific range; codec-specific default.
  Applies to whichever codec flag is in effect (`--codec` or
  `--dzi-format`).

**DZI / SZI-specific flags:**
- `--dzi-tile-size 256` (default)
- `--dzi-overlap 1` (default)
- `--dzi-format <jpeg|png>` — output codec for DZI / SZI; default `jpeg`.
  *(Shorthand: `--dzi-format` selects the codec for DZI / SZI targets
  rather than reusing `--codec`, because DZI / SZI accept only `jpeg`
  or `png` per the DZI / SZI spec — restricting the surface prevents
  invalid combinations like `--to dzi --codec avif`.)*

**`transcode` is REMOVED.** Migration is mechanical:
`wsitools transcode --container svs --codec jpeg --quality 85 IN OUT`
→ `wsitools convert --to svs --codec jpeg --quality 85 IN OUT`.

## 4. Tile-copy vs re-encode decision

```
target ∈ {cog-wsi, svs, tiff, ome-tiff}
  AND no --codec specified
  AND source is natively-tiled (not NDPI / OME-OneFrame)
  AND source codec is representable in target container
    → tile-copy (verbatim bytes; preserves v0.6 promise)

target ∈ {dzi, szi}                       → always re-encode
otherwise                                  → re-encode pipeline
```

The "codec representable in target container" check is a
conservative whitelist:

- COG-WSI: JPEG, AVIF, JP2K, WebP, JPEGXL, HTJ2K (per v0.6 spec).
- SVS: JPEG only (Aperio convention).
- TIFF: any codec opentile-go's `CompressionToTIFFTag` round-trips.
- OME-TIFF: same as TIFF.

If the source codec falls outside the whitelist (e.g. JPEGXL source
→ `--to svs` with no `--codec`), the command errors with a
suggestion to add `--codec jpeg`.

## 5. Architecture

### 5.1 Package layout

```
cmd/wsitools/
  convert.go                  # cobra command + --to dispatch
  convert_cogwsi.go           # existing; minor refactor to share helpers
  convert_tiff.go             # NEW; svs / tiff / ome-tiff (from transcode.go)
  convert_dzi.go              # NEW; --to dzi
  convert_szi.go              # NEW; --to szi (delegates to dzi + zip)
  convert_shared.go           # NEW; --codec / --quality / --workers parsing,
                              #      tile-copy-vs-re-encode decision helper
  transcode.go                # DELETED
  transcode_test.go           # DELETED (cases ported to convert_*_test.go)

internal/dzi/
  writer.go                   # Writer struct; manifest + tile-tree emission
  manifest.go                 # DZI XML manifest builder
  coords.go                   # level / tile coordinate math (mirrors
                              # opentile-go's internal/dzi/coords.go)

internal/szi/
  writer.go                   # wraps internal/dzi writer; ZIP central-dir
                              # build using archive/zip, store method
  scanprops.go                # builds scan-properties.xml from source Metadata()
```

### 5.2 Pipeline reuse

`internal/pipeline` (decode → resample → encode worker pool) is
unchanged. Per-target sinks plug in where `streamwriter` /
`cogwsiwriter` plug in today:

- cog-wsi / svs / tiff / ome-tiff sinks → `internal/tiff/streamwriter`
  or `internal/tiff/cogwsiwriter` (existing).
- dzi sink → `internal/dzi.Writer` (new).
- szi sink → `internal/szi.Writer` (new; wraps dzi.Writer).

The pipeline already enforces deterministic tile order, so DZI's
`<level>/<col>_<row>.<ext>` ordering is straightforward to emit.

### 5.3 DZI level generation

DZI requires the full `MaxLevel = ceil(log2(max(W, H)))` pyramid
down to a 1×1 tile. Source slides typically stop at L4. wsitools
synthesizes the missing tail by power-of-2 downsampling from L0
inside the existing resample stage (existing capability; same kernel
options as `downsample` subcommand).

Levels DZI uses but the source provides natively → decode-and-resample
from the closest native level (existing pipeline behaviour). Levels
finer than the source's L0 are not generated (no upscaling).

### 5.4 DZI tile geometry

- Tile size: `--dzi-tile-size` (default 256).
- Overlap: `--dzi-overlap` pixels added to each side of each tile
  (default 1). Edge tiles drop the overlap on the boundary side.
- Edge tiles: short on the right / bottom per the DZI spec — emit
  with actual content dimensions, not padded.
- Tile naming: `<name>_files/<level>/<col>_<row>.<ext>`. Note
  column-then-row per the DZI spec verbatim (matches opentile-go's
  reader).

### 5.5 SZI ZIP wrapper

- Single top-level directory inside the ZIP named `<basename>/`.
- All entries stored with `zip.Store` (method 0). No deflate.
  Preserves the mmap-aliasable read property opentile-go's SZI
  reader relies on.
- Entries written in DZI on-disk order: manifest first, then tiles
  in (level, col, row) order, then optional `scan-properties.xml`
  and `associated_images/`.
- Two emission modes:
  - **Streaming:** write entries directly into the ZIP as tiles
    complete (lower disk footprint, no temp dir).
  - **Spool:** write the DZI tree to a temp directory, then zip.
    Simpler but doubles disk use.
  - **v0.16 chooses streaming** to avoid the disk-footprint
    surprise (real WSIs are tens to hundreds of GB).

### 5.6 `scan-properties.xml`

Populated from the source slide's `opentile.Metadata`:

- `ScannerManufacturer`, `ScannerModel`, `Magnification`,
  `ScannerSerial`, `AcquisitionDateTime`, `ScannerSoftware`.

If the source doesn't expose a field (e.g. NDPI's
`ScannerManufacturer` is implicit), the field is omitted rather
than emitted as empty.

## 6. Test plan

### 6.1 Unit tests (no fixtures)

- `internal/dzi/coords_test.go` — `MaxLevel`, `LevelDims`, `GridDims`,
  edge-tile dimensions for representative sizes (CMU-1, scan_618_grundium).
- `internal/dzi/manifest_test.go` — XML round-trip; parser-compatible
  output verified against opentile-go's `internal/dzi/manifest.go`.
- `internal/szi/writer_test.go` — ZIP central-directory layout;
  every entry uses store method; opentile-go's SZI reader opens
  the result.
- `cmd/wsitools/convert_shared_test.go` — tile-copy-vs-re-encode
  decision matrix; codec whitelist per target; error messages for
  unsupported combinations.

### 6.2 Integration tests (gated by `WSI_TOOLS_TESTDIR`)

Per-target matrix using existing fixtures:

| Source fixture        | Target     | Mode       | Verifies                                 |
|-----------------------|------------|------------|------------------------------------------|
| CMU-1-Small-Region.svs| cog-wsi    | tile-copy  | bit-exact tile bytes (v0.6 promise)      |
| CMU-1-Small-Region.svs| cog-wsi    | re-encode  | --codec jpeg → re-encoded tiles          |
| CMU-1-Small-Region.svs| svs        | tile-copy  | output decodes; tile bytes match source  |
| CMU-1-Small-Region.svs| svs        | re-encode  | --codec jpeg --quality 80 → re-encoded   |
| CMU-1-Small-Region.svs| tiff       | tile-copy  | output decodes                           |
| CMU-1-Small-Region.svs| ome-tiff   | re-encode  | output passes OME-XML validation         |
| CMU-1-Small-Region.svs| dzi        | re-encode  | manifest valid; tile tree complete       |
| CMU-1-Small-Region.svs| szi        | re-encode  | opentile-go opens the result; round-trip |
| CMU-1.ndpi            | cog-wsi    | re-encode  | striped source → synthesized tiles       |
| CMU-1.ndpi            | dzi        | re-encode  | works on striped source                  |
| CMU-1.ndpi            | szi        | re-encode  | works on striped source                  |

DZI / SZI manifest validation: parse the emitted `.dzi` XML with
opentile-go's `internal/dzi.ParseManifest` and verify pyramid
dimensions match the source.

DZI tile-tree validation: walk every `<level>/<col>_<row>.<ext>`,
verify it decodes, and verify dimensions match DZI level / grid math.

### 6.3 Pixel-equivalence (non-byte-stable)

Per memory's pipeline-nondeterminism note: DZI / SZI tile bytes are
NOT expected to be byte-stable run-to-run. Output equivalence uses
`wsitools hash --mode pixel` on decoded pyramids; sums tile pixels
into a stable hash, ignoring tile-order writer nondeterminism.

### 6.4 Acceptance gate

- `make test` — all unit tests pass.
- `make goldens-byte-stable` — existing SVS goldens unchanged
  (cog-wsi tile-copy path).
- New `make goldens-dzi` (added in v0.16) — DZI / SZI pixel-hash
  goldens on the CMU-1 fixture.
- `make ci-full` (deferred per v0.15 roadmap; v0.16 adds the
  per-target sweep skeleton even if CI doesn't yet fetch fixtures).

## 7. CHANGELOG v0.16.0 entry

```markdown
## [0.16.0] — TBD

### BREAKING

- `transcode` subcommand removed. Use `convert --to <target>` with
  the same `--codec` / `--quality` / `--workers` flags.
  Migration is mechanical:
  - `wsitools transcode --container svs --codec jpeg IN OUT`
    → `wsitools convert --to svs --codec jpeg IN OUT`
  - `wsitools transcode --container tiff --codec webp IN OUT`
    → `wsitools convert --to tiff --codec webp IN OUT`

### Added

- `convert --to dzi` — DeepZoom pyramid output for OpenSeadragon
  and IIIF consumers. Defaults: 256×256 tiles, 1px overlap,
  JPEG Q=85.
- `convert --to szi` — Smart Zoom Image output: DZI pyramid wrapped
  in an uncompressed-stored ZIP archive with optional
  `scan-properties.xml` populated from source metadata.
- `convert --to {svs,tiff,ome-tiff}` — re-encode targets that
  subsume the former `transcode` flows.
- Tile-copy fast path now applies to `--to svs`, `--to tiff`,
  `--to ome-tiff` when no `--codec` is specified and the source is
  natively-tiled. Previously this was a cog-wsi-only optimisation.

### Internal

- `internal/dzi/` — pure-Go DZI writer (manifest + tile tree + level
  math). No cgo dependency.
- `internal/szi/` — SZI writer wrapping `internal/dzi` with an
  `archive/zip` store-method central directory.

### Deferred to v0.17

- ScaledStrips iterator (opentile-go v0.26) wired into the convert
  pipeline.
- libvips `dzsave` performance-comparison benchmark harness.

### Deferred indefinitely

- `--to dicom-wsi` — separate writer; tracked on roadmap.
- Narrow tile-copy path for DZI / SZI when `--dzi-overlap 0` +
  matching geometry + matching codec.
```
