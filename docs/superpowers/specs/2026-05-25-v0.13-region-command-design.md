# wsitools v0.13 — `region` subcommand

**Status:** Design approved 2026-05-25.

## 1. Goal

Add a `wsitools region` subcommand that extracts a rectangular pixel
region from a slide at a chosen pyramid level and writes the result
as a PNG. This is the first user-visible utility that consumes
opentile-go v0.25's `ReadRegion` family — exercises the new API end
to end and gives pathology consumers a single command for the
"openslide read_region + write PNG" workflow.

## 2. Scope

In scope for v0.13:

- New `cmd/wsitools/region.go` cobra subcommand.
- CLI flags:
  - `--level N` (required) — pyramid level index.
  - `--rect X,Y,W,H` OR all of `--x X --y Y --w W --h H` (one or the
    other; mutually exclusive). Coords + dims at the level's
    resolution.
  - `--image N` (optional, default 0) — for multi-image OME-TIFF.
  - `--format rgb|rgba` (optional, default rgb) — output pixel format.
  - `-o, --output PATH` (required) — output PNG path. Must end in
    `.png` (case-insensitive).
  - `-f, --force` (optional) — overwrite existing output file.
- PNG output via stdlib `image/png`.
- Bump opentile-go dep to v0.25.0.
- Update CHANGELOG.md and Version constant.

Out of scope (deferred):

- ReadRegionScaled support (`--l0-rect`, `--out-w`, `--out-h`).
  Scaled-region extraction lands when there's a real consumer
  (likely v0.14 alongside a `thumbnail` subcommand).
- JPEG / TIFF output. PNG is sufficient for the lossless decoded-
  pixel use case. JPEG is lossy and most consumers won't want
  re-encoded JPEG when they already have a pixel image; TIFF is
  bulky and most pathology pipelines already have tile-server
  infrastructure.
- Stdout output (`-o -`). Wait for demand.
- Multi-region batch mode. Current `for x; do wsitools region ...; done`
  shell loops handle this; a flag is unnecessary.

Non-goals:

- Changes to opentile-go (already at v0.25 with ReadRegion).
- Changes to other wsitools subcommands.

## 3. CLI surface

### Synopsis

```
wsitools region --level N (--rect X,Y,W,H | --x X --y Y --w W --h H)
                [--image N] [--format rgb|rgba] [-f] -o out.png slide
```

### Flag details

| Flag             | Type    | Required | Default | Description                                                                                  |
| ---------------- | ------- | -------- | ------- | -------------------------------------------------------------------------------------------- |
| `--level`        | int     | yes      | —       | Pyramid level index. Must be in `[0, len(slide.Levels()))`.                                  |
| `--rect`         | string  | one-of   | —       | Comma-separated `X,Y,W,H`. Four non-negative integers; W and H > 0.                          |
| `--x`            | int     | one-of   | —       | Level-coords X (top-left). Negative allowed (out-of-bounds left edge auto-white-filled).     |
| `--y`            | int     | one-of   | —       | Level-coords Y (top-left). Negative allowed.                                                 |
| `--w`            | int     | one-of   | —       | Width in level pixels. Must be > 0.                                                          |
| `--h`            | int     | one-of   | —       | Height in level pixels. Must be > 0.                                                         |
| `--image`        | int     | no       | 0       | For multi-image OME-TIFF. Must be in `[0, len(slide.Images()))`.                             |
| `--format`       | string  | no       | rgb     | `rgb` or `rgba`. RGBA pads with alpha=0xFF on every pixel.                                    |
| `-o, --output`   | string  | yes      | —       | Output file path. Must end in `.png` (case-insensitive).                                     |
| `-f, --force`    | bool    | no       | false   | Overwrite existing output file.                                                              |

`--rect` and `--x/--y/--w/--h` are MUTUALLY EXCLUSIVE. The subcommand
errors with `--rect and --x/--y/--w/--h are mutually exclusive; use
one form or the other` if any mix is provided.

If neither is fully specified (no `--rect` AND not all four
individual flags), the subcommand errors listing which flag(s) are
missing.

### Validation order (RunE)

1. Open slide; if fails, return wrapped open error.
2. Validate flag combination (rect vs individual; full set present).
3. Parse `--rect` if used; otherwise read individual flags.
4. Validate `--level` in range against opened slide; error
   `level %d out of range [0, %d)` otherwise.
5. Validate `--image` in range; same error pattern.
6. Validate `--output` extension == `.png`; error
   `--output: only PNG output is supported in v0.13 (got %q)`
   otherwise.
7. Validate output destination: if exists and `--force` not set,
   error `output exists; pass --force to overwrite`.
8. Read region via `slide.ImageReadRegion(...)`.
9. Encode + write PNG.
10. Print success message to stderr (slog INFO).

### Examples

```bash
# 512x512 patch from L0 at (1000, 1000)
wsitools region --level 0 --rect 1000,1000,512,512 -o patch.png slide.svs

# Same, explicit form
wsitools region --level 0 --x 1000 --y 1000 --w 512 --h 512 -o patch.png slide.svs

# Coarser level
wsitools region --level 2 --rect 256,256,512,512 -o patch.png slide.svs

# Out-of-bounds — partial extract, white-filled
wsitools region --level 0 --rect 99000,79000,2000,2000 -o edge.png slide.svs

# Multi-image OME-TIFF, second image's L0
wsitools region --image 1 --level 0 --rect 0,0,512,512 -o img1.png slide.ome.tiff

# RGBA output (alpha=0xFF everywhere)
wsitools region --level 0 --rect 0,0,256,256 --format rgba -o tile.png slide.svs

# Overwrite existing
wsitools region --level 0 --rect 0,0,512,512 -f -o existing.png slide.svs
```

### Errors users will see

```
$ wsitools region --rect 0,0,512,512 --x 100 ... -o p.png slide.svs
Error: --rect and --x/--y/--w/--h are mutually exclusive; use one form or the other

$ wsitools region --level 0 -o p.png slide.svs
Error: must specify either --rect or all of --x/--y/--w/--h

$ wsitools region --level 0 --x 100 -o p.png slide.svs
Error: must specify all of --x/--y/--w/--h (missing: --y --w --h)

$ wsitools region --level 99 --rect 0,0,512,512 -o p.png slide.svs
Error: level 99 out of range [0, 4)

$ wsitools region --level 0 --rect 0,0,512,512 -o p.bmp slide.svs
Error: --output: only PNG output is supported in v0.13 (got "p.bmp")

$ wsitools region --level 0 --rect 0,0,0,0 -o p.png slide.svs
Error: --rect: W and H must be positive (got W=0 H=0)
```

### Subcommand wiring

```go
package main

import (
    "fmt"
    "image"
    "image/png"
    "os"
    "strconv"
    "strings"

    opentile "github.com/wsilabs/opentile-go"
    "github.com/wsilabs/opentile-go/decoder"
    _ "github.com/wsilabs/opentile-go/decoder/all"
    _ "github.com/wsilabs/opentile-go/formats/all"
    "github.com/spf13/cobra"
)

var (
    regionLevel  int
    regionRect   string
    regionX      int
    regionY      int
    regionW      int
    regionH      int
    regionImage  int
    regionFormat string
    regionOutput string
    regionForce  bool
)

var regionCmd = &cobra.Command{
    Use:   "region [flags] <slide>",
    Short: "Extract a rectangular pixel region from a slide as PNG",
    Long:  `region extracts a rectangle of decoded pixels at a chosen pyramid level and writes it as a PNG file.`,
    Args:  cobra.ExactArgs(1),
    RunE:  runRegion,
}

func init() {
    regionCmd.Flags().IntVar(&regionLevel, "level", 0, "Pyramid level index (required)")
    regionCmd.Flags().StringVar(&regionRect, "rect", "", "Region rectangle as X,Y,W,H (level coords)")
    regionCmd.Flags().IntVar(&regionX, "x", 0, "Region X (level coords)")
    regionCmd.Flags().IntVar(&regionY, "y", 0, "Region Y (level coords)")
    regionCmd.Flags().IntVar(&regionW, "w", 0, "Region width (level pixels)")
    regionCmd.Flags().IntVar(&regionH, "h", 0, "Region height (level pixels)")
    regionCmd.Flags().IntVar(&regionImage, "image", 0, "Image index (multi-image OME-TIFF)")
    regionCmd.Flags().StringVar(&regionFormat, "format", "rgb", "Output pixel format: rgb|rgba")
    regionCmd.Flags().StringVarP(&regionOutput, "output", "o", "", "Output PNG path (required)")
    regionCmd.Flags().BoolVarP(&regionForce, "force", "f", false, "Overwrite existing output file")

    _ = regionCmd.MarkFlagRequired("level")
    _ = regionCmd.MarkFlagRequired("output")
    rootCmd.AddCommand(regionCmd)
}
```

### Flag-combination validation

```go
func resolveRect(cmd *cobra.Command) (x, y, w, h int, err error) {
    rectSet := cmd.Flags().Changed("rect")
    xSet := cmd.Flags().Changed("x")
    ySet := cmd.Flags().Changed("y")
    wSet := cmd.Flags().Changed("w")
    hSet := cmd.Flags().Changed("h")
    anyIndividual := xSet || ySet || wSet || hSet

    if rectSet && anyIndividual {
        return 0, 0, 0, 0, fmt.Errorf("--rect and --x/--y/--w/--h are mutually exclusive; use one form or the other")
    }
    if rectSet {
        return parseRect(regionRect)
    }
    if !anyIndividual {
        return 0, 0, 0, 0, fmt.Errorf("must specify either --rect or all of --x/--y/--w/--h")
    }
    // All four individual flags must be set.
    missing := []string{}
    if !xSet { missing = append(missing, "--x") }
    if !ySet { missing = append(missing, "--y") }
    if !wSet { missing = append(missing, "--w") }
    if !hSet { missing = append(missing, "--h") }
    if len(missing) > 0 {
        return 0, 0, 0, 0, fmt.Errorf("must specify all of --x/--y/--w/--h (missing: %s)", strings.Join(missing, " "))
    }
    if regionW <= 0 || regionH <= 0 {
        return 0, 0, 0, 0, fmt.Errorf("--w and --h must be positive (got W=%d H=%d)", regionW, regionH)
    }
    return regionX, regionY, regionW, regionH, nil
}

func parseRect(s string) (x, y, w, h int, err error) {
    parts := strings.Split(s, ",")
    if len(parts) != 4 {
        return 0, 0, 0, 0, fmt.Errorf("--rect: expected X,Y,W,H (4 comma-separated integers), got %q", s)
    }
    vals := make([]int, 4)
    for i, p := range parts {
        v, e := strconv.Atoi(strings.TrimSpace(p))
        if e != nil {
            return 0, 0, 0, 0, fmt.Errorf("--rect: not an integer: %q", p)
        }
        vals[i] = v
    }
    if vals[2] <= 0 || vals[3] <= 0 {
        return 0, 0, 0, 0, fmt.Errorf("--rect: W and H must be positive (got W=%d H=%d)", vals[2], vals[3])
    }
    return vals[0], vals[1], vals[2], vals[3], nil
}
```

### Core run logic

```go
func runRegion(cmd *cobra.Command, args []string) error {
    slidePath := args[0]

    // 1. Validate output extension early (cheap; fail fast).
    if !strings.HasSuffix(strings.ToLower(regionOutput), ".png") {
        return fmt.Errorf("--output: only PNG output is supported in v0.13 (got %q)", regionOutput)
    }

    // 2. Validate output destination.
    if _, err := os.Stat(regionOutput); err == nil && !regionForce {
        return fmt.Errorf("output exists; pass --force to overwrite (%q)", regionOutput)
    }

    // 3. Resolve rect from flags.
    x, y, w, h, err := resolveRect(cmd)
    if err != nil {
        return err
    }

    // 4. Open slide.
    slide, err := opentile.OpenFile(slidePath)
    if err != nil {
        return fmt.Errorf("opening slide %q: %w", slidePath, err)
    }
    defer slide.Close()

    // 5. Validate --image and --level against opened slide.
    images := slide.Images()
    if regionImage < 0 || regionImage >= len(images) {
        return fmt.Errorf("--image %d out of range [0, %d)", regionImage, len(images))
    }
    levels := images[regionImage].Levels
    if regionLevel < 0 || regionLevel >= len(levels) {
        return fmt.Errorf("--level %d out of range [0, %d)", regionLevel, len(levels))
    }

    // 6. Build decode options.
    var opts []opentile.DecodeOption
    switch regionFormat {
    case "rgb":
        opts = append(opts, opentile.WithFormat(decoder.PixelFormatRGB))
    case "rgba":
        opts = append(opts, opentile.WithFormat(decoder.PixelFormatRGBA))
    default:
        return fmt.Errorf("--format: expected \"rgb\" or \"rgba\", got %q", regionFormat)
    }

    // 7. Read the region.
    img, err := slide.ImageReadRegion(regionImage, regionLevel, x, y, w, h, opts...)
    if err != nil {
        return fmt.Errorf("reading region: %w", err)
    }

    // 8. Write PNG.
    if err := writeDecoderImagePNG(img, regionOutput); err != nil {
        return fmt.Errorf("writing PNG: %w", err)
    }

    fmt.Fprintf(os.Stderr, "wrote %s (%dx%d, %s)\n", regionOutput, img.Width, img.Height, regionFormat)
    return nil
}
```

### PNG encode helper

```go
// writeDecoderImagePNG converts a *decoder.Image to a stdlib image
// (NRGBA for RGBA, RGBA-channel-only via image.RGBA for RGB) and
// writes it to path via image/png.
func writeDecoderImagePNG(img *decoder.Image, path string) error {
    var stdimg image.Image
    if img.Format == decoder.PixelFormatRGBA {
        nrgba := &image.NRGBA{
            Pix:    img.Pix,
            Stride: img.Stride,
            Rect:   image.Rect(0, 0, img.Width, img.Height),
        }
        stdimg = nrgba
    } else {
        // RGB → unpack into image.RGBA (4 bytes/pixel; alpha=0xFF)
        rgba := image.NewRGBA(image.Rect(0, 0, img.Width, img.Height))
        for y := 0; y < img.Height; y++ {
            srcRow := img.Pix[y*img.Stride : y*img.Stride+img.Width*3]
            dstRow := rgba.Pix[y*rgba.Stride : y*rgba.Stride+img.Width*4]
            for x := 0; x < img.Width; x++ {
                dstRow[x*4+0] = srcRow[x*3+0]
                dstRow[x*4+1] = srcRow[x*3+1]
                dstRow[x*4+2] = srcRow[x*3+2]
                dstRow[x*4+3] = 0xFF
            }
        }
        stdimg = rgba
    }

    f, err := os.Create(path)
    if err != nil {
        return err
    }
    defer f.Close()
    return png.Encode(f, stdimg)
}
```

Note: `image.NRGBA` directly uses `decoder.Image.Pix` when format is
RGBA (zero-copy, fast). RGB requires a repack into a 4-byte stride.
The repack cost is bounded by `output_pixels × 4 bytes` — negligible
for any reasonable region size.

## 5. Testing strategy

### Unit tests (no slide fixture required)

`cmd/wsitools/region_test.go`:

- `TestParseRect` — table-driven: valid `"0,0,512,512"`, invalid
  `"0,0,512"` (only 3), `"0,0,abc,512"` (non-integer), `"0,0,0,512"`
  (zero W).
- `TestResolveRectMutuallyExclusive` — set both `--rect` and `--x`;
  expect "mutually exclusive" error.
- `TestResolveRectMissingIndividual` — set only `--x`; expect
  "missing --y --w --h" error.
- `TestResolveRectNoneSet` — expect "must specify either" error.

### Integration tests (fixture-gated via WSI_TOOLS_TESTDIR)

- `TestRegionBasicSVS` — run the binary against
  `CMU-1-Small-Region.svs`, level 0, 256x256 region, write PNG;
  decode the PNG with `image/png` and verify dimensions match.
- `TestRegionRGBAOutput` — same, with `--format rgba`; verify
  alpha=0xFF on every pixel.
- `TestRegionExtensionCheck` — `-o foo.bmp`; expect non-zero exit
  with the "only PNG" error.
- `TestRegionOverwriteWithoutForce` — pre-create the output file,
  run without `--force`; expect "output exists" error.
- `TestRegionOverwriteWithForce` — same, with `--force`; expect
  success.

### Coverage target

`cmd/wsitools/region.go` ≥80% line coverage. The flag-resolution
helpers are easy to test (pure functions); the encode helper is
exercised by the integration tests.
