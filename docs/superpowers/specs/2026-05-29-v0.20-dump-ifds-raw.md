# `dump-ifds --raw` design (v0.20)

**Status:** approved, ready for implementation plan
**Date:** 2026-05-29
**Scope:** expand `dump-ifds` with a tiffinfo-style full-tag dump per IFD.

## Goal

Add `--raw` (and `--raw-full`) flags to the existing `dump-ifds` subcommand so
that the command can emit every TIFF tag on every IFD, with names, types,
counts, and interpreted values where applicable. The current `dump-ifds`
output (slim, format-aware) remains the default and is unchanged.

Closes the v0.4-vintage roadmap note: "Not a full tiffinfo replacement — does
not dump every TIFF tag. A future `--raw` flag will expand this."

## Non-goals

- Editing tags (that's the planned `tagset` utility in Batch 3).
- Vendor-specific decoding of private tags beyond name lookup (Hamamatsu,
  Aperio, GeoTIFF). Unknown tags render as `<num> (unknown)` with raw bytes /
  decoded numeric value.
- Full EXIF parsing of EXIF SubIFDs. The EXIF IFD pointer (tag 34665) is
  reported as a long pointing to an offset; we do not recurse into it.

## CLI surface

```
wsitools dump-ifds [--json] [--raw] [--raw-full] <path>
```

- `--raw` — emit every TIFF tag (with type, count, value, interpretation).
  Composes with `--json` for machine-readable output.
- `--raw-full` — disable the smart truncation that `--raw` applies to long
  arrays, ASCII strings, and BYTE/UNDEFINED blobs. No-op when `--raw` is
  absent.
- Existing `--json` flag continues to drive output format; with `--raw`,
  the JSON shape changes to the per-entry array form described below.

Exit code, error messages, and file-open behavior are unchanged.

## Architecture

`internal/source/ifdwalk.go` gains a sibling entry point `WalkIFDsRaw` that
walks the same IFD chain as the existing `WalkIFDs` but also captures every
tag entry into a new `Entries []RawEntry` field on `IFDRecord`. The two
entry points share an internal `readIFDFull(captureAll bool)` helper:

- `captureAll=false` (slim) — current behavior. Typed fields populated; no
  `Entries`.
- `captureAll=true` (raw) — typed fields still populated (slim consumers
  on top of raw walk continue to work if ever needed), and every entry is
  appended to `Entries`.

The raw renderer lives in `cmd/wsitools/dump_ifds.go` alongside the slim
renderer. Tag-name dictionary and enum interpreters live in a new
`internal/tiff/tagnames.go`, kept alongside the existing tag constants in
`internal/tiff/tags.go`.

## Components

### `internal/source/ifdwalk.go` (modify)

New types:

```go
// RawEntry is one TIFF directory entry, captured verbatim.
type RawEntry struct {
    Tag         uint16
    Type        uint16 // 1..16; values outside are preserved as-is
    Count       uint64
    ValueOffset uint64 // raw offset/inline-value field bytes (LE-decoded)
    Raw         []byte // bytes pointed to (or the inline bytes); nil if unreadable
}
```

New / modified fields on `IFDRecord`:

```go
// ByteOrder is the file's byte order. Set by both WalkIFDs and WalkIFDsRaw.
ByteOrder binary.ByteOrder

// Entries is populated only by WalkIFDsRaw — one entry per tag, in
// directory order. nil from WalkIFDs.
Entries []RawEntry
```

New exported function:

```go
// WalkIFDsRaw is like WalkIFDs but additionally captures every directory
// entry into IFDRecord.Entries.
func WalkIFDsRaw(path string) ([]IFDRecord, error)
```

Internal refactor: today's `readIFD` becomes `readIFDFull(f, bo, bigTIFF, off, captureAll)`. The existing typed-field switch still runs unconditionally; when `captureAll` is true, a `RawEntry` is appended for every entry as it is read.

Malformed entries (count overflows file size, offset past EOF, etc.) produce a `RawEntry` with `Raw=nil`. The walker does not abort.

The 65536-entry cap and BigTIFF / classic header detection stay as today.

### `internal/tiff/tagnames.go` (new)

```go
// TagName returns the well-known name for a TIFF tag, or "" if unknown.
func TagName(tag uint16) string

// TypeName returns the TIFF type name (BYTE, SHORT, LONG, RATIONAL, ...) or
// "TYPE_<n>" for unknown types.
func TypeName(typ uint16) string

// TypeSize returns the number of bytes per element for a TIFF type. Returns
// 0 for unknown types.
func TypeSize(typ uint16) int

// InterpretEnum returns a friendly name for well-known enum tags (e.g.
// Compression=7 -> "JPEG"). Returns "" when no interpreter exists for the
// given tag.
func InterpretEnum(tag uint16, val uint64) string
```

### `cmd/wsitools/dump_ifds.go` (modify)

- New flags: `--raw`, `--raw-full`.
- Dispatch:
  - `--raw` absent → existing slim path, unchanged.
  - `--raw` present + `--json` absent → `renderRawText(os.Stdout, ifds, fullValues)`.
  - `--raw` present + `--json` present → `renderRawJSON(os.Stdout, ifds, fullValues)`.
- Help text update: drop the "A future `--raw` flag will expand this" sentence and document the new flags.

#### Text rendering

One block per IFD:

```
IFD 0 @ offset 0x10 (top-level, classic TIFF, byte-order=little-endian)
  254 (NewSubfileType)            LONG      count=1   value=0
  256 (ImageWidth)                SHORT     count=1   value=46920
  257 (ImageLength)               SHORT     count=1   value=33014
  258 (BitsPerSample)             SHORT     count=3   value=[8, 8, 8]
  259 (Compression)               SHORT     count=1   value=JPEG (7)
  262 (PhotometricInterpretation) SHORT     count=1   value=YCbCr (6)
  270 (ImageDescription)          ASCII     count=684 value="Aperio Image Library v11.2.1\r\n..."
  273 (StripOffsets)              LONG      count=24  value=[1234, 5678, 9012, ...] (... 16 more)
  322 (TileWidth)                 SHORT     count=1   value=240
  323 (TileLength)                SHORT     count=1   value=240
  347 (JPEGTables)                UNDEFINED count=574 value=<574 bytes: ff d8 ff e0 00 10 4a 46 49 46 ...>
  65080 (WSIImageType)            ASCII     count=8   value="baseline"
```

For SubIFDs the header line reads `IFD N @ offset 0x... (subIFD of IFD M)`.

Field widths: tag column padded so that the longest "<num> (<name>)" string in the IFD aligns; type column padded to longest type name; count column right-aligned.

#### JSON rendering

```json
[
  {
    "index": 0,
    "offset": 16,
    "is_bigtiff": false,
    "is_subifd": false,
    "parent_index": -1,
    "byte_order": "little",
    "entries": [
      {
        "tag": 256,
        "name": "ImageWidth",
        "type": 3,
        "type_name": "SHORT",
        "count": 1,
        "value": 46920
      },
      {
        "tag": 259,
        "name": "Compression",
        "type": 3,
        "type_name": "SHORT",
        "count": 1,
        "value": 7,
        "interpreted": "JPEG"
      },
      {
        "tag": 273,
        "name": "StripOffsets",
        "type": 4,
        "type_name": "LONG",
        "count": 24,
        "value": [1234, 5678, 9012, 13456, 17890, 22324, 26758, 31192],
        "truncated": true
      },
      {
        "tag": 347,
        "name": "JPEGTables",
        "type": 7,
        "type_name": "UNDEFINED",
        "count": 574,
        "value": "ffd8ffe000104a464946...",
        "truncated": true
      }
    ]
  }
]
```

- `name` is `""` (empty string) for unknown tags.
- `interpreted` is omitted when no interpreter exists for that tag.
- `truncated` is omitted when `value` represents the full content. Always `true` when content was truncated by smart-truncation rules; never `true` when `--raw-full` is set.
- `parent_index` is `-1` for top-level IFDs.
- `byte_order` is `"little"` or `"big"`.

#### Value decoding (shared helper)

`decodeValue(typ uint16, raw []byte, bo binary.ByteOrder) any` produces the
Go-native form used by both renderers:

| TIFF type      | Decoded form                                            |
|----------------|---------------------------------------------------------|
| BYTE (1)       | `[]byte` → hex string in JSON; hex in text              |
| ASCII (2)      | `string`, NUL-trimmed at first `\0`                     |
| SHORT (3)      | `uint64` scalar or `[]uint64` array                     |
| LONG (4)       | `uint64` scalar or `[]uint64` array                     |
| RATIONAL (5)   | `"num/den"` string or `[]string` array                  |
| SBYTE (6)      | `[]int8` → JSON array; comma-joined text                |
| UNDEFINED (7)  | same as BYTE                                            |
| SSHORT (8)     | `int64` scalar or `[]int64` array                       |
| SLONG (9)      | `int64` scalar or `[]int64` array                       |
| SRATIONAL (10) | `"num/den"` (signed)                                    |
| FLOAT (11)     | `float64` scalar or `[]float64` array                   |
| DOUBLE (12)    | `float64` scalar or `[]float64` array                   |
| IFD (13)       | `uint64` (offset)                                       |
| LONG8 (16)     | `uint64` (BigTIFF)                                      |
| SLONG8 (17)    | `int64` (BigTIFF)                                       |
| IFD8 (18)      | `uint64` (BigTIFF)                                      |
| unknown        | `[]byte` (raw); `type_name` = `TYPE_<n>`                |

Scalar-vs-array choice: `count==1` → scalar; `count>1` → array.

#### Smart truncation (default, off when `--raw-full`)

| Kind                         | Truncation                                    |
|------------------------------|-----------------------------------------------|
| Numeric arrays > 16 entries  | First 8 shown; text appends `(... N more)`    |
| ASCII > 200 chars            | Truncated with trailing `…` (text); `truncated:true` in JSON |
| BYTE/UNDEFINED, text mode    | First 32 bytes hex; `<N bytes: ...>` envelope |
| BYTE/UNDEFINED, JSON         | First 64 bytes as hex string                  |

`--raw-full` disables every cap above. Tag entries with `Raw=nil`
(unreadable) render as `value="<unreadable>"` in text and `null` in JSON,
with a `"error":"<reason>"` field added.

## Error handling

- Malformed directory entry (type=0, count overflow, bad offset): single
  entry rendered as `<num> (<name>) TYPE_<n> count=<n> value=<unreadable>`.
  Walker continues with the rest of the IFD.
- Truncated file mid-IFD: walker returns error (current behavior).
- Unknown tag type: render as `TYPE_<n>`, hex-dump the raw bytes.
- The 65536-entry-per-IFD cap remains and produces a hard error.

No new error paths to user; `--raw` is best-effort within an IFD.

## Testing

Fixture-gated integration tests (gated by `WSI_TOOLS_TESTDIR`; CI now runs
these on every push since v0.19):

1. `TestDumpIFDsRawCMU1Small` — `dump-ifds --raw sample_files/svs/CMU-1-Small-Region.svs`. Assert output contains tag names `ImageWidth`, `Compression`, `ImageDescription`, `Software`, `TileWidth`. Assert `Compression` line includes `JPEG (7)` and `PhotometricInterpretation` includes `YCbCr (6)`.
2. `TestDumpIFDsRawJSON` — `dump-ifds --raw --json` on the same file. Parse stdout as JSON; assert top-level is an array of length ≥ 6 (Aperio SVS layout: L0 + thumbnail + L1 + ... + label + macro). Find tag 256 in IFD 0; assert `name=="ImageWidth"` and `value` is an unsigned number.
3. `TestDumpIFDsRawNDPI` — CMU-1.ndpi. Assert at least one Hamamatsu private tag (range 65420–65458) appears with empty name and is rendered as `<num> (unknown)`.
4. `TestDumpIFDsRawTruncation` — assert `StripByteCounts` (or `TileByteCounts`) shows truncation marker `(... N more)` in text mode when count>16, and the marker disappears under `--raw-full`. Run the JSON variant and assert `"truncated":true` matches.

Unit tests (no fixture, run unconditionally):

5. `TestTagName` — table-driven: 10 known tags + 1 unknown (return `""`).
6. `TestTypeName` — 1..16 mapped; 0 and 99 return `TYPE_0`, `TYPE_99`.
7. `TestInterpretEnum` —
   - Compression: 1→`None`, 7→`JPEG`, 33003→`JPEG2000`.
   - PhotometricInterpretation: 2→`RGB`, 6→`YCbCr`.
   - NewSubfileType: 0→`""` (no decoration), 1→`reduced-resolution`, 5→`reduced-resolution|transparency-mask`.
   - PlanarConfiguration: 1→`chunky`, 2→`planar`.
8. `TestDecodeValueRational` — RATIONAL count=2 bytes `{1,0,0,0, 72,0,0,0, 1,0,0,0, 72,0,0,0}` (LE) → `["1/72","1/72"]`.
9. `TestDecodeValueAsciiNulTrim` — bytes `"hello\x00world\x00"` count=12 → `"hello"`.
10. `TestDecodeValueUnknownType` — type=99, 8 bytes → returns raw `[]byte`, `TypeName(99) == "TYPE_99"`.

## Release notes (v0.20)

- `dump-ifds --raw` — full per-tag dump with names + interpretation.
- `dump-ifds --raw --json` — same content, machine-readable.
